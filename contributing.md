# Contributing

Your contributions are always welcome! Please read these guidelines before opening a pull request.

## Guidelines

- Follow the [Awesome List Guidelines](https://github.com/sindresorhus/awesome/blob/main/contributing.md).
- Search existing entries before adding a new one to avoid duplicates.
- Make sure the resource is actually useful and related to Postman.
- Add your entry to the relevant section in alphabetical order when possible.
- Use the following link format: `- [Name](URL) - Short description.`
  - Description should start with a capital letter and end with a period.
  - Keep descriptions concise - one sentence is usually enough.
- Only submit resources that are actively maintained or historically significant.
- For libraries and tools, the GitHub repo must have had recent activity (within the past 2 years).
- Do not submit self-promotional content without clear community value.

## How to Add an Entry

1. Fork this repository.
2. Add your entry under the appropriate section in `README.md`.
3. Open a pull request with a short description of what you're adding and why it belongs here.

Thank you for making this list more awesome!
